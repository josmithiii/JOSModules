◆ prepareToPlay()

virtual void FaustModule::prepareToPlay ( double  samplingRateHz,
int  maxSamplesPerBlock 
)
virtual

Tell the module to prepare for playing.

The prepareToPlay(samplingRateHz) method must be called before useful output can be obtained from processBlock().

When playback has finished, releaseResources() is called to release any persistent resources allocated by prepareToPlay().

Parameters
samplingRateHzthe audio sample rate expected for all audio buffers processed. Filter coefficients depend on the sampling rate, for example.
maxSamplesPerBlockthe maximum number of samples in each AudioBuffer passed to processBlock(). Smaller buffer sizes are fine, but not larger, when it is used. (Some modules allocate persistent arrays of this size to achieve best performance.)
See also
releaseResources, processBlock

Reimplemented in Freeverb, and Zitarev.

Called for each successive block of audio data.

After calling prepareToPlay(), this function is called (in the audio thread) each time the audio destination (such as audio playback hardware) requests another block of audio data.
